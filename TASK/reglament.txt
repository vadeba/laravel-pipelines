    1. Именование переменных
Каждая переменная должна нести свою смысловую нагрузку, например:
Если input отвечает за колонки таблицы - называть cols ($_GET[’cols’], $cols и т.д. и т.п. ) 
Если название переменной составное - использовать camelCase - например переменная “статус пользователя” -> $userStatus
P.S. используйте английский язык – гуглите
переменные $i, $j, $k … применимы только в итераторах (цикл for и т.п.)
    2. php код начинается строго с <?php и закрывается ?> - шорттег <? НЕ ИСПОЛЬЗОВАТЬ!
Исключение - атомарные операции, пример:
<td><?=$i?> <- данная конструкция заменяет <?php echo $i;?>

позволяются небольшие операции - арифметические\конкатенации
(<?=$i*$j?> , <?=$i.$j?> ….)

    3. в php.ini флаг error_reporting должен быть только E_ALL (для openserver -> дополнительно - конфигурация — php(версия)
display_errors=On так же

    4. Разрыв кода - имеется в виду вёрстка - отдельно, код - отдельно, использовать echo, print и т.д. для вывода вёрстки ЗАПРЕЩЕНО!

echo "<div class='test $class'>$i</div>"; - НЕЛЬЗЯ
?> <div class="test <?=$class?>"><?=$i?></div> <?php- НУЖНО!

<?php if([condition]) {
echo "<p>..........$variable…..</p>";
} ?> -- ПЛОХО

<?php if([condition]) { ?>
<p>..........<?=$variable?>…..</p>;
<?php } ?> -- ХОРОШО
    5. Все формы сопровождать подстановкой введенных ранее значений
<form method="POST">
	<input type="text" name="surname" value="<?=$_POST['surname'] ?? ''?>">

если GET - то заменить!
данное выражение заменяет следующий код:
<?php if(isset($_POST['surname'])) { echo $_POST['surname'] } else{ echo ''; } ?>

    6. Отступы! Весь код должен иметь правильный отступ каждого блока, что в вёрстке - что в PHP коде
<div>
<?php if($a) {
		[код]
		foreach (...) {
			[код]
		}
	}
?>
</div>	
	
<div>
<?php if($a) {
[код]
foreach (...) { [код] }
	}
	?>
</div>

Отступы делаются табуляцией - в phpStorm есть волшебная комбинация клавиш Ctrl+Alt+L 

Строго типизируем все аргументы если это возможно (будет применимо в функциях, методах классов – входные параметры, выходной тип)
